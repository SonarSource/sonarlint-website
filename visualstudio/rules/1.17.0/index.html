There are no new rules in this version of SonarLint for Visual Studio. However, a couple of rules have been improved to filter some false positives, check the release notes for the list of modified rules. Furthermore, the data flow analysis engine was significantly improved. Relational operators are now evaluated, so we can report on always true or false conditions in many more cases than before. The below code fragment shows a couple of cases that we can recognize with this release. 

int a = ...
int b = ...

if (a == b)
{
  if (a > b) {}                             // Noncompliant, condition is always false
  if (a != b) {}                            // Noncompliant, condition is always false
  if (a.Equals(b)) {}                       // Noncompliant, condition is always true
  if (object.Equals(a, b)) {}               // Noncompliant, condition is always true
}
