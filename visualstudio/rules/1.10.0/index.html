SonarLint for Visual Studio version 2.0

  • Release date: March 17, 2016
  • Release notes
  • Go to latest version

With SonarLint for Visual Studio 2.0 you can now establish a connection to a SonarQube server and bind your Visual Studio solution to a SonarQube project. This operation automatically updates the rulesets of the solution and attaches the required Roslyn analyzers to the solution. This connected mode requires the SonarQube C# plugin 5.0+ on the SonarQube server. Check out a short introduction on how to use this feature in the FAQ.

The 2.0 version is the output of a tight collaboration between Microsoft and SonarSource. The SonarSource team would like to offer warm thanks to the Microsoft Visual Studio ALM team in Cambridge.

SonarLint relies on the new SonarAnalyzer C# and VB.NET Nuget packages, which were previously shipped as a single SonarLint Nuget package. To avoid any further confusion from now on we are refering to the Nuget packages by the name SonarAnalyzer (for C#, for VB.NET), and to the Visual Studio extension by SonarLint.
