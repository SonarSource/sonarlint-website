SonarLint for Visual Studio - version 1.6.0

  • Release date: December 9, 2015
  • Release notes
  • Go to latest version

We have added 11 new C# rules in this version of SonarLint. We improved Rule S2971 to handle even more cases where LINQ call chains can be simplified. Additionally, we disabled some rules to improve the out-of-the-box experience, and fixed some false positives reported by the community. Finally, we grouped the rules into logical categories, so the Visual Studio RuleSet editor can now list the rules by categories. Here are the newly added rules since version 1.5.0:

  • Method overloads with default parameter values should not overlap - Rule S3427
  • Method calls should not resolve ambiguously to overloads with "params" - Rule S3220
  • "base.Equals" should not be used to check for reference equality in "Equals" if "base" is not "object" - Rule S3397
  • Attribute, EventArgs, and Exception type names should end with the type being extended - Rule S3376
  • "ConfigureAwait(false)" should be used - Rule S3216
  • Culture should be specified for String operations - Rule S1449
  • Overflow checking should not be disabled for "Enumerable.Sum" - Rule S2291
  • The length returned from a stream read should be checked - Rule S2674
  • "async" methods should not return "void" - Rule S3168
  • "FIXME" tags should be handled - Rule S1134
  • "TODO" tags should be handled - Rule S1135
