SonarLint for Visual Studio - version 1.1.0

  • Release date: August 7, 2015
  • Release notes
  • Go to latest version

There are 88 rules in SonarLint for Visual Studio version 1.1.0. Thirteen are new since version 1.0.0:

  • Fields should not be initialized to default values - Rule S3052
  • "Explicit" conversions of "foreach" loops should not be used - Rule S3217
  • Classes directly extending "Object" should not call "base" in "GetHashCode" or "Equals" - Rule S3249
  • Unused method parameters should be removed - Rule S1172
  • "value" parameters should be used - Rule S3237
  • "GC.SuppressFinalize" should not be invoked for types without destructors - Rule S3234
  • Anonymous delegates should not be used to unsubscribe from Events - Rule S3244
  • Redundant parentheses should not be used - Rule S3235
  • Doubled prefix operators "!!" and "~~" should not be used - Rule S2761
  • Multiple "OrderBy" calls should not be used - Rule S3169
  • Delegates should not be subtracted - Rule S3172
  • Inner class members should not shadow outer class "static" or type members - Rule S3218
  • "interface" instances should not be cast to concrete types - Rule S3215

One rule has been dropped due to too many false positives:

  • Methods and properties that don't access instance data should be static - Rule S2325 (version 1.0.0)
