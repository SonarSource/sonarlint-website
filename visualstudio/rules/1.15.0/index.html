SonarLint for Visual Studio version 2.4

  • Release date: June 28, 2016
  • Release notes for SonarLint
  • Release notes for C#
  • Go to latest version

SonarLint for Visual Studio version 2.4 adds two new rules and some improvements to the product. The improvements include the extension of Rule S1172 to report on parameters whose value is never used because of assignments on all execution paths. Rule S1854 has been fixed to not report on variables that are captured in lambdas. Also, the issue location of Rule S1854 and Rule S1481 have been improved to not report on the same declaration. Finally, here are the new rules since version 2.3:

  • Null pointers should not be dereferenced - Rule S2259
  • Conditions should not unconditionally evaluate to "true" or to "false" - Rule S2583
