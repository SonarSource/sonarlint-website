SonarLint for Visual Studio version 2.2

  • Release date: May 4, 2016
  • Release notes for SonarLint
  • Release notes for C#
  • Download Extension

The connected mode has been improved to provide notification when the quality profile changed and the solution needs resync with the SonarQube server. Also, the SonarQube exclusion MSBuild properties can be edited through a nicer UI now. SonarLint version 2.2 adds 8 new rules to the product. Furthermore, 17 rules have been improved or fixed to not report false positives, handle more cases, have better severity settings or just simply to have better issue messages. Here are the new rules since version 2.1:

  • Methods with "Pure" attribute should return a value - Rule S3603
  • Events should be invoked - Rule S3264
  • Private fields only used as local variables in methods should become local variables - Rule S1450
  • Variables should not be checked against the values they're about to be assigned - Rule S3440
  • Unassigned members should be removed - Rule S3459
  • One-way "OperationContract" methods should have "void" return type - Rule S3598
  • "ServiceContract" and "OperationContract" attributes should be used together - Rule S3597
  • "params" should not be introduced on overrides - Rule S3600
