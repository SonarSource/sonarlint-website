SonarLint for Visual Studio - version 1.2.0

  • Release date: August 24, 2015
  • Release notes
  • Go to latest version

There are 90 rules in SonarLint for Visual Studio version 1.2.0. Two are new since version 1.1.0:

  • Default parameter values should not be passed as arguments - Rule S3254
  • The simplest possible condition syntax should be used - Rule S3240

This is the first version of SonarLint with automatic code fixes. List of rules with code fixes:

  • The ternary operator should not return the same value regardless of the condition - Rule S2758
  • "catch" clauses should do more than rethrow - Rule S2737
  • Redundant parentheses should not be used - Rule S3235
  • Field-like events should not be virtual - Rule S2290
  • Doubled prefix operators "!!" and "~~" should not be used - Rule S2761
  • "GC.SuppressFinalize" should not be invoked for types without destructors - Rule S3234
  • Silly bit operations should not be performed - Rule S2437
  • Multiple "OrderBy" calls should not be used - Rule S3169
  • Useless "if(true) {...}" and "if(false){...}" blocks should be removed - Rule S1145
  • Generic parameters not constrained to reference types should not be compared to "null" - Rule S2955
  • Fields that are only assigned in the constructor should be "readonly" - Rule S2933
  • Boolean checks should not be inverted - Rule S1940
  • Boolean literals should not be redundant - Rule S1125
  • Redundant casts should not be used - Rule S1905
  • "ThreadStatic" should not be used on non-static fields - Rule S3005
  • Default parameter values should not be passed as arguments - Rule S3254
  • The simplest possible condition syntax should be used - Rule S3240
