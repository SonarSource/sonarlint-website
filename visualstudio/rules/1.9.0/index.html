Ten new rules have been added to this version of SonarLint for Visual Studio. Furthermore, some rules have been improved to handle more cases or to filter false positives. Also, two bugs have been found and fixed that caused exceptions in rules when analysing invalid code. Altogether there are 153 C# rules in SonarLint for Visual Sudio. Check the new rules since version 1.8.0:

  • Interfaces with colliding, inherited members should explicitly redefine interface members - Rule S3444
  • Type examining methods should be avoided on "System.Type" instances - Rule S3443
  • Inappropriate casts should not be made - Rule S1944
  • Optional parameters should be passed to "base" calls - Rule S3466
  • Non-flags enums should not be used in bitwise operations - Rule S3265
  • "[DefaultValue]" should not be used when "[DefaultParameterValue]" is meant - Rule S3451
  • Right operands of shift operators should be integers - Rule S3449
  • Limit nesting depth of control flow statements in functions - Rule S134
  • Redundant property names should be omitted in anonymous classes - Rule S3441
  • Redundancies in declarations - Rule S3257

Rules that have been reworked:

  • Rule S2333 has been changed to not report on virtual keywords, and at the same time it now looks for redundant unsafe keywords.
  • Rule S1698 now doesn't report on cases when the Equals override is known to perform a reference equality check.
  • Rule S2184 also reports on returned integer divisions and lambda return values.
  • Rule S2737 not only checks the last catch block, but all redundant ones.

Finally, I'd like to thank the contribution from Pepijn van de Kamp, who implemented Rule S134. Great job!
