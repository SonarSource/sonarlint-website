SonarLint for Visual Studio - version 1.5.0

  • Release date: November 16, 2015
  • Release notes
  • Go to latest version

We added 12 C# rules to SonarLint. 8 of these rules will have parameters in the future, but for the time being the parameters cannot be set, so they are using reasonable default values. There are 120 C# rules now in SonarLint for Visual Studio. Also, we have continued bringing VB.NET rules to SonarLint for Visual Studio, we added 10 more (4 that also comes for C#), so there are 30 VB.NET rules altogether. Here are the main changes since version 1.4.0:

C# rules:

  • Methods should not have too many parameters - Rule S107
  • "switch" statements should not have too many "case" clauses - Rule S1479
  • Class names should comply with a naming convention - Rule S101
  • Method name should comply with a naming convention - Rule S100
  • Methods should not be too complex - Rule S1541
  • Expressions should not be too complex - Rule S1067
  • Files should not have too many lines - Rule S104
  • Lines should not be too long - Rule S103

C# and VB.NET rules:

  • Strings should not be concatenated using '+' in a loop - Rule S1643
  • Public constant members should not be used - Rule S2339
  • Statements should be on separate lines - Rule S122
  • Optional parameters should not be used - Rule S2360

VB.NET rules:

  • Fields should be private - Rule S2357 (Already existed for C#)
  • "Do" loops should not be used without a "While" or "Until" condition - Rule S2340
  • Event names should not have 'Before' or 'After' as a prefix or suffix - Rule S2349
  • "Exit Select" statements should not be used redundantly - Rule S2350
  • "Exit" statements should not be used - Rule S3385
  • Line continuations should not be used - Rule S2354

Finally, we fixed some false positives and bugs. Thank you for your feedback. In version 1.5.0, we have fixed issues that were reported by Jesse Houwing, Wim Reymen, Neil M, huserben, Niek Jannink, and blairmcg.
