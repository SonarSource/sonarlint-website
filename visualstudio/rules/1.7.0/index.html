SonarLint for Visual Studio - version 1.7.0

  • Release date: January 25, 2016
  • Release notes
  • Go to latest version

We have added 5 new rules with code fixes to this version of SonarLint for Visual Studio. Furthermore, all known bugs and false positives have been fixed. And finally, the activation of rules have been checked and several rules have been disabled by default. These changes result in less noise and a better overall developer experience. The new rules since version 1.6.0:

  • Overriding members should do more than simply call the same member in the base class - Rule S1185
  • "string.Format()" should not be called without placeholders - Rule S3457
  • "string.ToCharArray()" should not be called redundantly - Rule S3456
  • "ToString()" calls should not be redundant - Rule S1858
  • Unused private types or members should be removed - Rule S1144

False positives that have been filtered in this version:

  • Rule S1854 does not report inside "try" blocks
  • Rule S1172 does not report on methods that have attributes
  • Rule S2934 does not produce false-positives with chained type constraints
  • Rule S1172 does not report on methods assigned to delegate or non-private methods
  • Rule S2930 has been reworked to only report on a select few "IDisposable" types and factory methods
  • Rule S1134 and Rule S1135 only report on full word matches
