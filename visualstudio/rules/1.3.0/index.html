SonarLint for Visual Studio - version 1.3.0

  • Release date: October 6, 2015
  • Release notes
  • Go to latest version

There are 101 rules in SonarLint for Visual Studio version 1.3.0. Many new since version 1.2.0:

  • "params" should be used on overrides - Rule S3262
  • Inheritance list should not be redundant - Rule S1939
  • Redundant modifiers should not be used - Rule S2333
  • Literal suffixes should be upper case - Rule S818
  • Format strings should be passed the correct number of arguments - Rule S2275
  • "base()" constructor calls should not be explicitly made - Rule S3253
  • Static fields should appear in the order they must be initialized - Rule S3263
  • Caller information arguments should not be provided explicitly - Rule S3236
  • Trivial properties should be auto-implemented - Rule S2292
  • "Type.IsAssignableFrom()" should not be used to check object type - Rule S2219
  • Generic type parameters should be co/contravariant when possible - Rule S3246

Also, we have added a few more code fixes, and fixed issues with the batch fixers. List of rules with new code fixes:

  • Inheritance list should not be redundant - Rule S1939
  • Redundant modifiers should not be used - Rule S2333
  • Literal suffixes should be upper case - Rule S818
  • "base()" constructor calls should not be explicitly made - Rule S3253
  • "Type.IsAssignableFrom()" should not be used to check object type - Rule S2219
  • "GetHashCode" should not reference mutable fields - Rule S2328
  • Property assignments should not be made for "readonly" fields not constrained to reference types - Rule S2934
  • "Explicit" conversions of "foreach" loops should not be used - Rule S3217
  • Methods should not be empty - Rule S1186
  • Empty statements should be removed - Rule S1116

Finally, there are some bug fixes in this release. We are grateful if you report issues on GitHub, and we do our best to fix issues by the upcoming release. In version 1.3.0, we have fixed issues that were reported by Terje Sandstrom, Jesse Houwing, sergn-n, jbfraser1, Duncan Pocklington. I hope I haven't missed anyone. Thank you.
