SonarLint for Visual Studio - version 1.8.0

  • Release date: February 15, 2016
  • Release notes
  • Go to latest version

Seven new rules have been added to this version of SonarLint for Visual Studio. Furthermore, several rules have been reworked to not report on controversial cases, or to remove known false positives or bugs. Also, issue messages have been clarified for a few rules so they should be understandable without additional description. The new rules since version 1.7.0:

  • "==" should not be used when "Equals" is overridden - Rule S1698
  • Parameters with "[DefaultParameterValue]" attributes should also be "[Optional]" - Rule S3450
  • "[Optional]" should not be used with "ref" or "out" - Rule S3447
  • Optional parameters should have the same default value in overrides - Rule S1006
  • Possible loss of fraction - Rule S2184
  • "default" clause of a "switch" should not be empty - Rule S3532
  • Implementations should be provided for "partial" methods - Rule S3251

Rules that have been reworked:

  • Rule S3236 does not report on cases when the passed value is also a parameter with caller info attribute.
  • Rule S2219 now reports correctly on IsAssignableFrom calls.
  • Rule S3217 only reports on foreach statements on generic collections.
  • Rule S1449 now also handles StringComparison arguments.
  • Rule S2971 doesn't report false positives on ToList() calls.
